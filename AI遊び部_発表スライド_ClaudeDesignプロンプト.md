=== ここからプロンプト ===

社内部活「AI遊び部」の LT 発表用スライドを作ってください。テーマは、AI を使った健康管理ツール「Health-Manager」の紹介です。参加者を"健康の沼"に引きずり込むのが裏テーマなので、部活らしくゆるく楽しいノリでお願いします。

## 発表の前提

- 場: 社内部活「AI遊び部」。参加者はエンジニア中心だが、ターミナルに不慣れな人もいる
- 全体は1時間。**最初の約15分がこのスライドを使った発表**、**残り約45分は参加者と一緒に手を動かすハンズオン**
- 言語: すべて日本語
- トーン: フランクで前向き。専門用語には必ずカッコで短い日本語補足を付ける(例: 「推定1RM(1回で挙げられる最大重量の目安)」)

## 発表タイトル

**AI is 最強のHealthパートナー ── 健康は"AIと回す"時代へ**

サブタイトル: 「AI遊び部 presents / あなたを健康の沼に引きずり込む Health-Manager 入門」

## スライド構成(見出しは厳守。各見出しはセクション扉+本文で構成)

### 表紙
- タイトル・サブタイトル・登壇者名(空欄でOK)。ゴリラのマスコットを大きく配置。

### 1. 肉体美を育む3条件 ＝ 栄養・運動・記録。続かないのは「記録」だけ
- 体づくりは「栄養(食べる)」「運動(動かす)」「記録(残す)」の3つが揃って初めて回る、と図で示す
- 栄養と運動はやる気で始められるが、**記録だけが地味で面倒**。だから最初に脱落するのは記録、という気づきを強調
- 補足: 筋肉を伸ばす原則である漸進性過負荷(少しずつ負荷を増やす)は、前回の記録がないと成立しない=記録は最重要なのに一番続かない、という矛盾を提示

### 2. その記録の摩擦を、AIがまるごと肩代わりする
- 「記録の面倒」を具体的に分解: 食べたものの栄養計算、種目・重量・レップ数の転記、グラフ化、続けるモチベ維持
- これを全部 AI が代行する、というビフォーアフターを対比で見せる
- ユーザーがやることは「話す/スクショを渡す」だけ。あとは AI が代筆・計算・可視化する
- 結論: 人は"記録"から解放され、"追い込む(栄養と運動)"ことだけに集中できる

### 3. Health-Manager:話すだけで計画・記録・レポートまでAIが担当
- Health-Manager とは: Claude と GitHub リポジトリを組み合わせ、筋トレ・食事・睡眠・歩数・水分を一元管理するテンプレート、と一言で紹介
- 月次サイクルを PDCA(計画→実行→振り返り→改善)として図解:
  - 毎月1日「今月のメニューを作って」と話す → 選択式の質問(目標・練度・頻度・場所・食事スタイル)に答える → 1ヶ月分の筋トレ&食事計画と、毎日の記録ページが自動で用意される
  - 毎日: トレーニング/食事の結果を Claude に伝える(文章・スクショ・アプリの CSV/PDF なんでもOK)
  - 毎晩「本日のメニュー終了」→ グラフ付きの日次ふりかえりレポートが生成される
  - 毎週日曜「今週のメニュー終了」→ 週次レポート(部位別ボリューム・体重推移など)
  - 翌月は前月の実績を踏まえた計画に自動でアップデート
- 安心ポイント(1枚): メニュー・食事は出典付きの種目データベース(78種目)・食材データベース(52食材)からのみ生成され、AI が数値をでっち上げない設計であることを、信頼できるツールとして強調

### 4. Claude Artifact ── AIが作る、あなただけの可視化ダッシュボード
- Claude Artifact を軽く説明: 会話の中から、その場で動く Web ページやアプリを AI が生成してくれる機能、と噛み砕く
- Health-Manager では、日次・週次レポートがこの Artifact として**インタラクティブなダッシュボード**になる
- 中身の例: 部位別のトレーニング量、体重の推移、推定1RM(1回で挙げられる最大重量の目安)の伸び、摂取カロリー・たんぱく質の達成度など
- スマホでもPCでも開ける「自分専用の健康ダッシュボードが一瞬で手に入る」体験価値を見せる(ダッシュボード風のイメージ図を配置)

### 5. ハンズオン(構築手順 → ライブデモ)※ここが本番・約45分
このセクションは手順を1つずつ丁寧に。コマンドは大きめのコードブロックで、コピーしやすく表示すること。

- **扉**: 「ここから一緒に手を動かします」+ 所要イメージ
- **用意するもの**(1枚・表形式): GitHub アカウント(無料)/ Claude アカウント(Pro プラン以上を推奨)/ パソコン(Mac または Windows)
- **Step 1: 自分専用リポジトリを作る**
  - テンプレートの「Use this template」→「Create a new repository」
  - **Visibility は必ず Private**(体重・体脂肪率などの健康情報を扱うため)。ここを一番大きく警告
- **Step 2: 道具を2つ入れる**
  - GitHub CLI: Mac は `brew install gh` / Windows は `winget install --id GitHub.cli`
  - Claude Code: `npm install -g @anthropic-ai/claude-code`(`npm` が無ければ先に Node.js を入れる)
- **Step 3: GitHub と接続**
  - `gh auth login`(基本すべて Enter → ブラウザで8桁コードを入力して Authorize)
- **Step 4: リポジトリを手元に持ってくる**
  - `gh repo clone <あなたのユーザー名>/<リポジトリ名>`
  - `cd <リポジトリ名>`
- **Step 5: 初期化スクリプトを実行**
  - `bash scripts/setup.sh`(記録用ラベルの作成・当月データファイルの生成・テンプレ内リンクの書き換えが自動で走る)
  - `git add -A && git commit -m "chore: initial setup" && git push`
- **Step 6: Claude を起動して最初の計画を作る**
  - `claude` と打って起動 → 「今月のメニューを作って」と話しかける → 選択式の質問に答えるだけ
- **ライブデモの流れ**(登壇者が画面で実演。1〜2枚):
  1. 「今月のメニューを作って」→ ヒアリング → 1ヶ月分の計画と毎日の記録ページが一括で作られる様子
  2. その日の結果を渡す(スクショ or テキスト)→「本日のメニュー終了」→ グラフ付き日次レポート & Claude Artifact のダッシュボードが出てくる様子
- **つまづきポイント Tips**(1枚): 間違えて Public で作ったら Settings→Danger Zone で Private に変更 / `gh: command not found` はターミナル再起動 / スマホ派は Claude.ai + GitHub 連携でもOK

### 6. まとめ:継続はAIに任せて。あなたはただ追い込もう
- 3条件のうち一番の脱落原因だった「記録」を AI が肩代わりする → だから続く、という締め
- あなたは栄養と運動に全集中すればいい
- 「さあ、一緒に健康沼へ」と参加を促す前向きなクロージング。ゴリラが応援しているビジュアル

## デザイン要件(重要)

- **配色**: ベースは薄い肌色(ライトベージュ〜ピーチ系の淡い背景)。アクセントに元気なポップな黄色を効かせる。全体は明るく親しみやすいトーンで統一
- **マスコット**: ゆるくて可愛いゴリラのイラストを全体で使う。マッチョだけど表情はゆるふわ。セクションごとに違うポーズで登場させる(例: ダンベルを持つ・スマホで記録する・レポートを眺める・応援する)。要所要所に小さく配置し、表紙とまとめでは大きく
- **アニメーション**: 積極的に活用する。要素のフェードイン/スライドイン、セクション切り替えのトランジション、数字のカウントアップ、ゴリラのちょっとした動き(ゆれる・ダンベルを上げる等)を入れて、飽きさせない
- **フォント/形状**: 丸ゴシック系の親しみやすい書体、角丸のカードやふきだしを多用
- **図解重視**: 文字を詰め込みすぎない。3条件・PDCAサイクル・ビフォーアフター・手順フローは図やアイコンで見せる

## 分量の目安

- 発表パート(表紙・見出し1〜4・6)は合計15分で話せる量 = 15〜18枚程度に収める
- ハンズオン(見出し5)は手順を飛ばさず、1ステップ1枚 + デモ流れ = 10枚前後
- コマンドはコピペしやすい大きさで正確に(誤字厳禁)
