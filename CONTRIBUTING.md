# CONTRIBUTING.md — テンプレートへの貢献ガイド

自分のリポジトリで検証した種目・食材・スキル改善をテンプレートへ還元する場合のガイド。本テンプレートを fork し、PR を作成してください(あなたの記録用リポジトリからは PR できません。テンプレートとの git 上の関係が無いためで、これは仕様です)。

## 最重要原則

**あなたの個人データ(体組成・実績・plans/・results/・data/)を PR に含めないこと。** PR に含めてよいのは `.claude/`・`references/` の初期DB部分・`scripts/`・`.github/`・ドキュメントのみ。

## 種目DB(exercise-db.md)への追加基準

- 種目名・対象部位(主働/協働)・場所・必要器具・難易度・未経験者向け説明・推奨レップ帯・参考URL の全列を埋める
- 推奨レップ帯には出典脚注を付ける(programming.md の既存出典に対応しない場合は新規出典を追記)
- **参考URLは実在確認済みのもののみ**。確認日を `(確認日: YYYY-MM-DD)` 形式で併記する。優先ソース: 公認団体(ACE 等)・ExRx・公的機関。架空URL・未確認URLは即リジェクト
- 追加先は各セクション(自宅/ジム/屋外)の初期DB表。冒頭の登録数・セクション見出しの件数も更新する

## 食材DB(nutrition-db.md)への追加基準

- 100gあたり kcal・P・F・C は**日本食品標準成分表(版数明記)または商品の栄養成分表示**に基づく値のみ。記憶・概算値は不可
- 標準1単位・コンビニ入手区分(◎/△/×)・出典列を埋める
- 一般的なスーパー・コンビニで入手可能な食材に限定する

## Skills / Agents の変更基準

- Claude Code と Claude.ai の**両環境で動作する**こと(環境依存処理は「環境判定」節で分岐)
- CLAUDE.md の絶対規則(生成ソースのDB限定・架空URL禁止・数値根拠必須・医療行為禁止 等)と矛盾しないこと
- frontmatter は `name` と `description` のみ(Agent Skills 標準)
- スキル本文を変更した場合、README の同期チェックリスト(zip 再配布)に影響する旨を PR 説明に書く

## PR チェックリスト

- [ ] 個人データを含んでいない
- [ ] URL は全て実在確認済み・確認日併記
- [ ] 数値は出典付き
- [ ] 件数表記(DB冒頭・セクション見出し)を更新した
- [ ] 変更理由と検証方法(実運用で何日使ったか等)を PR 説明に記載した
