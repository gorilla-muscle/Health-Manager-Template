# 日次 Issue 本文テンプレート

monthly-plan が発行する日次 Issue の本文形式。`{{ }}` を実値に置換する。

## タイトル

- トレーニング日: `[{{YYYY-MM-DD}}] Day{{NN}} - {{分割名}}({{対象部位}})`
  - 例: `[2026-07-12] Day12 - Pull(背中・二頭)`
- 休養日: `[{{YYYY-MM-DD}}] Day{{NN}} - Rest`

## ラベル

- トレーニング日: `training` `nutrition` `month:{{YYYY-MM}}`
- 休養日: `rest` `nutrition` `month:{{YYYY-MM}}`

## フォーマット規則

- **本日のメニューの並び順(固定)**: トレーニング日は メトリクス目標 → 食事目標 → トレーニング目標、休養日は メトリクス目標 → 食事目標 の順で記載する。この順序を変えない
- **メトリクス目標の数値根拠**: 目標歩数・睡眠時間・摂取水分量は `programming.md` §6 を根拠に設定する(推測値を書かない・絶対規則3)
- **体調チェック**: GitHub のタスクリスト記法(`- [ ]`)で記載する。チェック済み(`- [x]`)は「問題あり」を意味する。ユーザーが記入する欄なので発行時は未チェックのままにする
- **セクション間隔**: 可読性のため、各セクション(見出し・表・メモブロック)の区切りに `<br>` を1行単独で挿入する。`<br>` の前後には空行を置く(GitHub のレンダリングで確実に間隔が入る)
- 見出し直後のコンテンツとの間には空行を入れる(CommonMark 準拠)
- Issue 本文を生成・更新する際は本テンプレートの `<br>` 配置をそのまま踏襲する
