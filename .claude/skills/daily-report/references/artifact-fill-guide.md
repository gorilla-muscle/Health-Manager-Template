# artifact-fill-guide.md — HTMLテンプレートの埋め方ガイド

`artifact-template.html` は日次レポート HTML の完全固定 scaffold。**このテンプレートファイル自体は絶対に上書き・編集しない**（レポート生成の作業中に `artifact-template.html` へ変更を加えない）。**CSS / レイアウト / SVG 描画方式 / カラーパレット / セクションの並び順は絶対に変更しない**。必ず Read → Write で `results/YYYY-MM/daily/YYYY-MM-DD.html` へ丸ごとコピーし、以下のプレースホルダと繰り返しブロックだけを埋めて生成する。テンプレート本体のデザインを変えてよいのは、ユーザーが明示的にテンプレート改修を依頼したときのみ。

## 生成手順

1. `Read` でテンプレート全体を取得
2. `Write` で対象日の HTML(例: `results/2026-07/daily/2026-07-13.html`)へコピー
3. 以下の対応表に従って `{{PLACEHOLDER}}` を実値へ、`<!-- REPEAT:XXX start -->` 〜 `<!-- REPEAT:XXX end -->` の中身を項目数だけ複製、`<!-- IF:XXX start/end -->` は条件成立時のみ残す
4. テンプレート先頭のガイドコメント(`<!-- daily-report アーティファクト HTML 完全固定テンプレート ... -->`)は削除
5. 埋め終わった HTML を Artifact ツールで publish

**CSS/骨格の変更禁止**: `<style>` ブロック、`<main>` 直下の `<header>` `<section>` `<footer>` の順序と数、各 `<section>` の `<h3><span class="eyebrow-num">01|02|03|04|05</span>...` は毎日同じ。

### 達成判定の許容幅(摂取カロリー・たんぱく質)

摂取カロリー・たんぱく質は目標ぴったりにするのが困難なため、以下の許容幅内なら**達成(OK)扱い**とし、警告色・不足警告を出さない。

| 項目 | 達成(OK)の条件 | 未達(警告)の条件 |
|---|---|---|
| 摂取カロリー | 実績 ≧ 目標 − 200 kcal | 実績 < 目標 − 200 kcal |
| たんぱく質 | 実績 ≧ 目標 − 10 g | 実績 < 目標 − 10 g |

- **OK 時(許容内)**: `{{KPI_KCAL_CLASS}}` / `{{KPI_P_CLASS}}` は **`pos`(緑=達成)** を入れる(空欄や `warn` にしない)。DETAIL は「計画 194g に対し 8g 少なめ・許容範囲内(達成率 95.9%)」のように**達成である旨**を書き、「不足」という語は使わない。`{{KCAL_WARNING_HTML}}` は空文字にし、たんぱく質バーも不足強調しない
- **未達 時(許容超の不足)**: `neg`(赤系)で不足量を明示し、必要なら `{{KCAL_WARNING_HTML}}` に補足を書く
- 許容幅は**不足方向のみ**に適用する。カロリー超過(特に減量期)は従来どおり評価する
- 達成率(`{{KCAL_PCT}}`・`{{P_PCT}}` 等)の数値そのものは実測どおり表示する(許容幅で数値を書き換えない。色と文言だけを許容幅で判定する)
- 基礎代謝(BMR)を下回る等の健康上の注意は許容幅と独立。該当時は許容内でも `{{KCAL_WARNING_HTML}}` に注意を残す

**判定の具体例**(この通りに色分けする):

| 目標 | 実績 | 差 | 判定 | CLASS | DETAIL 例 |
|---|---|---|---|---|---|
| たんぱく質 194 g | 186 g | −8 g | 許容内(≦10g)→ **達成** | `pos`(緑) | `計画 194g に対し 8g 少なめ・許容範囲内(達成率 95.9%)` |
| たんぱく質 194 g | 180 g | −14 g | 許容超 → 未達 | `neg`(赤) | `計画 194g に対し 14g 不足(達成率 92.8%)` |
| 摂取カロリー 2,770 | 2,600 | −170 | 許容内(≦200)→ **達成** | `pos`(緑) | `計画 2,770 に対し 170 kcal 少なめ・許容範囲内(達成率 93.9%)` |
| 摂取カロリー 2,770 | 1,891 | −879 | 許容超 → 未達 | `neg`(赤) | `計画 2,770 に対し 879 kcal 不足(達成率 68.3%)` |

### 02 目標 vs 実績(SVG ドーナツと PFC バー)

**ドーナツの dasharray 計算**(円周 = 2π × 半径72 = 452.39 で固定):

```
{{KCAL_DASH}} = 452.39 × ({{KCAL_PCT}} / 100)  ※ 100% 超過時も 452.39 でクリップ
```

例:
- `{{KCAL_PCT}}` = `68.3` → `{{KCAL_DASH}}` = `452.39 × 0.683` = `309.03`
- `{{KCAL_PCT}}` = `100` → `{{KCAL_DASH}}` = `452.39`
- `{{KCAL_PCT}}` = `115` → `{{KCAL_DASH}}` = `452.39`(超過時もクリップ)

| プレースホルダ | 内容 | 例 |
|---|---|---|
| `{{KCAL_PCT}}` | カロリー達成率(%・小数第1位) | `68.3` |
| `{{KCAL_DASH}}` | 上式で算出 | `309.03` |
| `{{KCAL_ACTUAL}}` | 摂取実績(kcal・カンマ区切り) | `1,891` |
| `{{KCAL_DELTA_TEXT}}` | 差分の一文 | `計画 2,770 kcal に対し 879 kcal 不足` |
| `{{KCAL_WARNING_HTML}}` | 警告文(空欄可・HTML 埋め込み可) | `⚠ 基礎代謝(BMR = 何もしなくても消費するカロリー)は約 1,889 kcal。<br>今日の摂取量はこのラインぎりぎりで、これ以上減ると筋肉が落ちやすくなります。` |

### 03 体組成推移(直近30日・計測がある系列のみ)

`data/metrics/YYYY-MM.md`(+ 前月ファイル)から直近30日分を読み、**体重・体脂肪率・骨格筋の3系列**について推移を SVG 折れ線で描画する。各系列は**独立**に判定:

- 計測が **2件以上**ある系列 → `<!-- IF:TREND_XXX start/end -->` を残し、`<!-- IF:TREND_XXX_INSUFFICIENT start/end -->` を削除
- 計測が **1件以下**の系列 → 逆に `_INSUFFICIENT` 側だけを残す
- 系列間で判定は独立(体重は描けるが骨格筋はデータ不足、等の混在可)

`XXX` は `WEIGHT` / `BF` / `SM` の3種(体重・体脂肪率・骨格筋)。

**SVG 座標系の計算**(全系列共通・viewBox は `0 0 320 100`):

- X: 記録日を古い→新しい順に等間隔で並べる。i 番目(0-indexed)の x 座標 = `20 + i * (280 / (N - 1))` (N は記録件数)
- Y: 値を `ymin..ymax` にスケール。ymin = min − 0.5・ymax = max + 0.5(体重・骨格筋)/ min − 0.3・max + 0.3(体脂肪率)。Y 座標 = `10 + (1 - (v - ymin) / (ymax - ymin)) * 80`(上ほど値が大きい)
- polyline の points 属性は `x1,y1 x2,y2 ...` 形式の空白区切り文字列を `{{TREND_XXX_POLYLINE}}` に入れる
- circle は `<!-- REPEAT:TREND_XXX_POINT start/end -->` で件数分だけ複製する

**各系列のプレースホルダ**(3系列とも同じ命名パターン・`XXX = WEIGHT | BF | SM`):

| プレースホルダ | 内容 | 例 |
|---|---|---|
| `{{TREND_XXX_LATEST}}` | 最新記録値(単位なしの数値) | `89.9` |
| `{{TREND_XXX_DELTA}}` | 期間内の変化量(符号付き) | `−1.6 kg` / `+0.2 %` |
| `{{TREND_XXX_POLYLINE}}` | 折れ線 SVG points 属性値 | `20,42 90,38 160,50 230,35 300,44` |
| `{{TREND_XXX_XSTART}}` | X 軸左端の日付ラベル | `7/12` |
| `{{TREND_XXX_XEND}}` | X 軸右端の日付ラベル | `7/15` |
| `{{TREND_XXX_YMIN}}` `{{TREND_XXX_YMAX}}` | Y 軸下限・上限 | `89.0` / `92.0` |
| `{{TREND_XXX_NEEDED}}`(不足時のみ) | あと何回で描けるか | `1` |

**口調のルール**（他人行儀・事務的な書き方は禁止）:

- 敬語（です・ます）は保ちつつ、「〜ですね！」「すごいです！」「いきましょうね！」のように **「！」で感情・熱量を表現する**。本人のすぐ隣で声をかける、あたたかく前向きなトーンにする
- 「！」は 1 文に 1 つ程度を目安に（連発して軽薄にならないように）。冷たい言い切り（「〜すること。」「〜が必要である。」）や、突き放した評論調にしない
- **持ち上げは必ず具体的な数値・行動とセット**にする（中身のない称賛だけにしない）。良かった点は具体的に認め、要改善点は責めずに明日の具体行動へ翻訳する。最後に明日の最優先1点を必ず示す
- 数値は一般語で表現し、内部用語（ファイル名・規則番号・DB 名）は書かない
