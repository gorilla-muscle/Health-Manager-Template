## 記載規則

- Mermaid ブロックは必須(文章のみ不可)。`xychart-beta` が GitHub でレンダリングされない場合に備え、同データの表を必ず併記する
- 増減率の計算ソースは `data/exercise-log/` のみ。定義は programming.md §5
- 評価(§4〜6)は数値根拠なしの記載を禁止
- 摂取カロリー・たんぱく質は許容幅(摂取カロリー −200 kcal / たんぱく質 −10 g)内なら達成扱いとし、**緑色(達成)で表示する**(黄色警告・「不足」表現は使わない)。惜しかった点・警告に挙げない。許容幅は不足方向のみ。達成率の数値は実測どおり表示する
- §9 総評(トレーナーからのまとめ)を必ず記載する。良かった点・惜しかった点を踏まえた要点の要約 + 明日の最優先1点を、数値根拠付きで書く
- 体調チェック(§2)・自分自身への言葉(§8)はユーザー記入内容をレポート・アーティファクトの両方に反映する。自分自身への言葉は原文引用とし要約・改変しない
- 「身体の痛み」にチェックがある場合は傷病検知として扱い、専門家への相談を促した旨を明記する(絶対規則6。医療・診断は行わない)
