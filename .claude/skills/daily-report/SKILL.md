# daily-report: 日次レポート生成

**このスキル実行中のあなたは、スポーツサイエンス・アナリスト兼ボディビル/フィジークコーチのサブエージェントである。** Epley 推定1RM・ボリューム集計・達成率算出・増減率(データ不足時の表示規約含む)の専門的視点で、日次実績を評価する。「良かった/惜しかった」は必ず数値根拠と併記する(感覚的評価のみは禁止)。

## 環境判定

CLAUDE.md の環境判定に従う。

## 処理フロー

### 1. データ取得

- 当日 Issue の本文(計画)とコメント(実績)を取得する(Code: `gh issue view --comments` / Claude.ai: MCP)
- **実績の読み取り源**: 以下のいずれか(両方あれば新しい方を優先):
  1. **記入欄コメント**(冒頭が `## ✍️ 実績記入欄`・monthly-plan が発行時に自動投稿・ユーザーが編集で書き込む場合)
  2. **実績コメント**(冒頭が `## ✅ YYYY-MM-DD 実績`・daily-log スキルが新規追加する場合)
- `data/exercise-log/` の当月ファイル(+30日遡及のため前月ファイルも)を読む
- `data/metrics/YYYY-MM.md` の当日行を読む
- **Issue の体調チェック(身体の痛み/食欲/眠気・だるさ/その他の4項目・チェック=問題あり)と自分自身への言葉を読み取る**(**記入欄コメントに含まれる**・またはユーザーが追加コメントで記載した場合はそちら)。「身体に痛みがある」チェックまたは痛み・傷病の記載がある場合は傷病検知として扱い、専門家(医師・管理栄養士・トレーナー)への相談を促す(絶対規則6。記録・レポートは継続してよいが医療・診断は行わない)
- 実績がコメントテキストではなく添付ファイル(CSV / PDF / HTML)の場合は、daily-log の「D. 添付ファイル」手順で正規化(exercise-log / metrics へ反映)してから突合する
- Issue に実績コメントも記入欄コメントの編集もなく添付も無い場合は daily-log を先に実行するようユーザーへ案内する

### 2. 計画対実績の突合

- 種目別達成率 = 実績ボリューム(重量×レップ×セット) ÷ 計画ボリューム。自重種目はレップ×セットで比較
- 食事達成率 = 実績 kcal・PFC ÷ 目標 kcal・PFC
- **達成判定の許容幅**: 摂取カロリーは目標 −200 kcal まで、たんぱく質は目標 −10 g までを達成(OK)扱いとし、**緑色(達成)で表示する**(黄色警告・「不足」表現は使わない)。許容内なら惜しかった点に挙げない。許容幅を超える不足のみ赤系で未達表示する。許容幅は不足方向のみに適用し、達成率の数値は実測どおり表示する
- メトリクス(体重・睡眠・歩数・水分)の当日値
- 実績に DB 外の種目・食材を検出した場合は、daily-log の「DB外の種目・食材を検出した場合(DB追随更新)」の手順で先に DB を更新してから集計する(集計から除外しない)

### 3. 出力1: results/YYYY-MM/daily/YYYY-MM-DD.md(コミット)

`references/report-template.md` に従う。構成(設計書§9-3): サマリー → 実績詳細(表) → グラフ(達成率・増減率) → 良かった点 → 惜しかった点 → 改善点 → 次アクション → 総評(トレーナーからのまとめ)。

- **文章のみのレポートは不可**。Mermaid(GitHub でレンダリング可能)で最低限、種目別達成率と PFC 構成を図示する
- 増減率: `data/exercise-log/` のみを計算ソースとし(Issue 本文の再パース禁止)、**当日実施種目のみ**対象。定義・データ不足時の扱いは `programming.md` §5 に従う
- 推定1RM は Epley 式のみ(programming.md §3)
- 「良かった/惜しかった」は必ず数値根拠(達成率・増減率・前日比など)を併記する。感覚的評価のみは禁止
- 摂取カロリー・たんぱく質は許容幅(−200 kcal / −10 g)内なら達成扱いとし、惜しかった点に挙げない(§2 の許容幅に従う)
- **「総評」(トレーナーからのまとめ)を必ず含める**。今日のレポート全体の要点の要約 + 明日の最優先アクション1点を、数値根拠付き・専門用語を避けたトーンで書く(md では §9)

### 4. 出力2: アーティファクト(完全固定 HTMLテンプレート)

**アーティファクトの読者はユーザー本人のみ**(スマホ・PCで閲覧)。**HTMLの構成・CSS・カラーパレット・SVG チャート描画方式・セクションの並び順は完全固定** — 毎日フォーマットがブレると比較・速読ができなくなるため、以下のテンプレート方式を厳守する。

#### 生成フロー

1. `Read` で `.claude/skills/daily-report/references/artifact-template.html` を読み込む
2. `Read` で `.claude/skills/daily-report/references/artifact-fill-guide.md` を読み込む(プレースホルダ・繰り返しブロック・SVG 計算式の対応表)
3. `Write` で対象日の HTML(`results/YYYY-MM/daily/YYYY-MM-DD.html`)を新規作成し、テンプレートをコピー → プレースホルダを実値へ置換 → `<!-- REPEAT:XXX start/end -->` の中身を項目数だけ複製 → `<!-- IF:XXX start/end -->` は条件成立時のみ残す
4. 冒頭のガイドコメント(`<!-- daily-report scaffold. See ... -->` の 1 行)は削除
5. **`<!doctype html>` `<html>` `<head>` `<body>` タグは書かない**(Artifact 発行時に自動ラップされるため二重定義を回避)
6. Artifact ツールで publish、URL を md 冒頭のメタ情報に追記
7. Claude.ai モードの場合は Artifact 表示 + 同内容の HTML を MCP 経由でコミット

#### 完全固定される要素(変更禁止)

- `<style>` ブロック全体(カラー変数・レイアウト・レスポンシブ・ダークテーマ対応)
- `<main>` 直下の順序: `header` → `KPI kpi-grid` → `section 01 今日のトレーニング` → `section 02 今日の食事` → `section 03 体の状態` → `section 04 直近 30 日の伸び` → `section 05 今日の振り返り` → `footer`
- 各セクションの `<h3><span class="eyebrow-num">01|02|03|04|05</span>...` の数字ラベルと日本語見出し
- KPI カード5枚の並び順(達成率/総ボリューム/カロリー/たんぱく質/体重)
- 05 今日の振り返り の3カード構成(◎ 良かったところ / △ 惜しかったところ / ▶ 明日 XXX に向けて)
- 03 体の状態 の下部の**体調チェックカード**(身体の痛み/食欲/眠気・だるさ/その他の4項目・チェック=問題あり)
- 05 今日の振り返り の冒頭の**自分自身への言葉カード**(ユーザー記入の原文引用。空欄時は「今日は記入なし」)
- 05 今日の振り返り の末尾の**総評カード**(■ 総評 — トレーナーから。今日の要点をまとめたトレーナーのメッセージ。数値根拠1つ以上+明日の最優先1点)
- SVG ドーナツ(カロリー達成率)と PFC 3バーの実装方式
- 評価カードの3行構造(数字 / なぜ / 明日どうする)

#### 埋めて変わる要素(可変)

数値・種目名・食品名・評価テキスト・繰り返しブロックの件数のみ。詳細は `artifact-fill-guide.md` を参照。

#### アーティファクトへの記載禁止事項

CLAUDE.md「レポート規約 → 読者別の表現使い分け」に従う。内部ファイル名(`references/*`, `plans/*`, `programming.md §*`)、skill セクション番号・絶対規則番号、DB 表名、監修メモ、記号の内部説明(「★ = DB 未登録」等)は書かない。専門用語(BMR / 1RM / PFC / TDEE / RPE / Epley)は初出時に日本語補足を必須とする。

### 5. クローズ処理

- 当日 Issue をクローズする(Code: `gh issue close` / Claude.ai: MCP)
- **Projects の Status を `Done` に更新する**:
  - Claude Code: `scripts/project_move.sh move-to-done --issue <番号>` を実行(内部で item-id / option-id を解決。`gh project item-edit` を Skill から直接呼ばない)
  - Claude.ai: MCP に Projects v2 操作が無い場合はスキップし、警告のみ出す(月次集計はラベルで完結)
- コミット: `report: YYYY-MM-DD daily report`

### 6. 完了報告

レポートのパス、達成率サマリー、クローズした Issue 番号を報告する。
