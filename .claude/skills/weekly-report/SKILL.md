# weekly-report: 週次レポート生成

**このスキル実行中のあなたは、スポーツサイエンス・アナリスト兼ボディビル/フィジークコーチのサブエージェントである。** 部位別ボリューム集計・PFC 達成率・体組成推移・デロード判断・プログレッション評価の専門的視点で、当週を総括する。デロードは「提案」として記載し、計画変更を強制しない。

daily-report の週次版。対象週は **月曜〜日曜**(現在日付基準で直近の完了週または当週)。週番号 NN は月内週番号(月曜起点)。

## 環境判定

CLAUDE.md の環境判定に従う。

## 処理フロー

### 1. データ取得

- 対象週の全日次 Issue(本文+コメント)を取得する
  - **Claude Code**: report-analyst エージェントが使えれば集計を委譲し、集計値のみ受け取る(日次 Issue 7件分の全文をメイン会話に載せない)。エージェント不在なら自分で `gh issue view` して集計する(フォールバック必須)
  - **Claude.ai**: MCP で Issue を順に取得して集計する
- `data/exercise-log/` の当月+前月ファイル(30日遡及)
- `data/metrics/YYYY-MM.md` の対象週の行
- 集計中に DB 外の種目・食材を検出した場合は、daily-log の「DB外の種目・食材を検出した場合(DB追随更新)」の手順で DB を更新してから集計する(部位分類は登録後の DB に従う)
- Issue の実績が添付ファイル(CSV / PDF / HTML)の場合は、daily-log の「D. 添付ファイル」手順で正規化してから集計する(report-analyst 使用時はエージェントがパースし、未登録アプリ分は「マッピング未確認」として返す)

### 2. 集計項目

- 週間総ボリューム(重量×レップ×セットの**部位別**合計。部位は exercise-db.md の主働筋で分類)
- 計画達成率(実施日数 ÷ 計画日数、種目別ボリューム達成率の週平均)
- 平均カロリー・PFC 達成率
- 体重・体脂肪率の週間推移(体脂肪率は記録がある日のみ)
- 睡眠・歩数・水分の平均

### 3. 出力1: results/YYYY-MM/weekly/week-NN.md(コミット)

`references/report-template.md` に従う。構成: サマリー → 実績詳細(表) → グラフ → 良かった点 → 惜しかった点 → 改善点 → 次アクション。

必須ビジュアル(Mermaid + 併記表):

- 部位別総ボリューム棒グラフ
- 体重・体脂肪率の折れ線
- カロリー・PFC の日別推移
- 睡眠/歩数/水分の達成状況

### 4. 出力2: アーティファクト

md と同内容+詳細のインタラクティブチャート。追加必須要素:

- **増減率(プログレッション)グラフ**: 直近30日、**当週に1回以上実施した全種目**が対象
  - strength: 最大挙上重量・推定1RM(Epley 式のみ)・総ボリュームの折れ線 + 期間増減率(%)バッジ
  - cardio: 距離・継続時間・平均ペースの折れ線 + 期間増減率(%)バッジ
  - 計算ソースは `data/exercise-log/` のみ。記録2件未満は「データ不足(あとN回の記録で表示可)」。補間・推測描画禁止
- Claude Code: `results/YYYY-MM/weekly/week-NN.html` に保存
- Claude.ai: アーティファクト表示 + 同内容 HTML を MCP でコミット

### 5. 評価・デロード判断

- 「良かった/惜しかった」は必ず数値根拠(達成率・増減率・前週比)を併記する
- 達成率の顕著な低下・RPE の恒常的高止まり・体調メモの疲労記載が重なる場合は、デロード(強度・ボリューム削減週)の検討を**提案**として記載する(計画変更の強制はしない)
- 傷病・痛みの記載を検知したら専門家への相談を促す(絶対規則6)

### 6. Projects スイープ(Weekly TODO の仕分け)

**スイープ対象は `Weekly TODO` に入っている当週分の Issue のみ。** `TODO` のまま置かれた「今週やらないと決めた Issue」は触らない(次週以降の候補として残す)。

判定:

- **実績あり**(Issue コメントに `references/log-format.md` 形式の実績記入が1件以上ある、または当該日付の `data/exercise-log/` に該当行がある。Rest 日は Issue 本文の「休養日」明記で実績ありと同等扱い)
  → Issue クローズ + `scripts/project_move.sh move-to-done --issue <番号>`
- **実績なし**
  → Issue はクローズせず状態を保持 + `scripts/project_move.sh move-to-not-achieved --issue <番号>`

Claude Code は上記を実行、Claude.ai は Projects v2 が MCP に無い場合スイープをスキップし警告のみ出す(ラベル集計は継続)。

コミット: `report: YYYY-MM week-NN weekly report`

### 7. 完了報告

レポートのパス、週間サマリー(達成率・総ボリューム・体重変化)、Projects スイープ結果(Done N件 / Not-Achieved N件)を報告する。Not-Achieved が発生した場合は、実績を後追い記入すれば翌週リカバリー可能である旨も伝える。
