# health-manager-template

筋トレ・食生活・睡眠・歩数・水分量を、Claude(Rules / Skills / Agents)と GitHub リポジトリで一元管理するシステムのテンプレート。

> ⚠️ **必ず Private リポジトリとして作成してください。** このシステムは体組成・健康情報という個人情報を扱います。

## 導入(3ステップ)

> 🔰 **ターミナルに慣れていない方へ**: 図解レベルまで噛み砕いた手順を、この README 内の [はじめてのセットアップ(詳細ガイド)](#はじめてのセットアップ詳細ガイド) にまとめています。初めての方はまずそちらを上から順に進めてください。

1. このページ上部の **「Use this template」→「Create a new repository」** をクリックし、**Visibility は必ず Private** を選択して自分のリポジトリを作成する(fork や clone ではなくテンプレートから作成することで、あなたの記録は完全にあなたのリポジトリ内で完結します)
2. 作成したリポジトリを手元に clone し、初期化スクリプトを実行する:

   ```bash
   gh auth login   # 未認証の場合
   bash scripts/setup.sh
   ```

   ラベル一括作成・当月データファイルの生成・Issue テンプレート内リンクの自分のリポジトリへの書き換えが行われます
3. Claude に接続する(下記「Claude との接続」)

## はじめてのセットアップ(詳細ガイド)

このセクションは、**ターミナル(黒い画面)をほとんど使ったことがない方**でも Health-Manager を使い始められるように書かれています。上から順番に進めてください。

- 所要時間の目安: 30〜60分
- 途中でわからなくなったら: 末尾の「困ったときは」を見てください

### 0. これは何をするためのもの?

Claude(AI アシスタント)があなた専属のトレーナー兼栄養士のように動き、

- 月初に、あなたに合った **1ヶ月分の筋トレメニューと食事計画** を作る
- 毎日の記録場所(GitHub の「Issue」というメモページ)を **1ヶ月分自動で用意** する
- 毎晩・毎週、記録をもとに **グラフ付きのふりかえりレポート** を作る

これを実現する仕組み一式が、このテンプレートに入っています。

> ⚠️ 医療に関する注意: このシステムは医療行為・診断を行いません。持病や痛みがある方は、先に医師・専門家に相談してください。

### 1. 用意するもの

| 必要なもの | 説明 | 料金 |
|---|---|---|
| GitHub アカウント | 記録を保存する場所。メールアドレスがあれば作れます | 無料 |
| Claude アカウント | AI 本体。**Pro プラン以上を推奨**(Claude Code を使うため) | 有料プランあり |
| パソコン | Mac または Windows | — |

#### 1-1. GitHub アカウントを作る(持っていない場合)

1. ブラウザで https://github.com/signup を開く
2. メールアドレス・パスワード・ユーザー名を入力して登録する
3. 届いた確認メールのコードを入力して完了

#### 1-2. Claude アカウントを作る(持っていない場合)

1. https://claude.ai を開いて登録する
2. プランは Pro 以上を推奨(このあと使う「Claude Code」が利用できます)

### 2. 自分専用のリポジトリ(記録置き場)を作る

「リポジトリ」とは、GitHub 上のフォルダのようなものです。あなたの計画と記録はすべてここに保存されます。

1. ブラウザで、GitHub 上のこのテンプレートのリポジトリページを開く
2. 緑色の **「Use this template」** ボタン → **「Create a new repository」** をクリック
3. 次の通りに入力する:
   - **Repository name**: `my-health-manager` など好きな名前
   - **Visibility**: 必ず **Private** を選ぶ ← ここが一番大事!
4. **「Create repository」** をクリック

> 🔒 **なぜ Private 必須?** 体重・体脂肪率などの健康情報を記録するためです。Public(公開)にすると全世界から見えてしまいます。もし間違えて Public で作ってしまったら、「困ったときは」の項を見てください。

### 3. ターミナルを開いてみる

ここから少しだけ「ターミナル」を使います。ターミナルは、パソコンに文字で指示を出すためのアプリです。**表示されたコマンドをコピーして、貼り付けて、Enter キーを押す**。やることはこれだけです。

- **Mac の場合**: `command + スペース` を押して「ターミナル」と入力し、Enter
- **Windows の場合**: スタートボタンを右クリック →「ターミナル」(または「PowerShell」)をクリック

白か黒の文字だけの画面が開けば成功です。

### 4. 道具を2つインストールする

#### 4-1. GitHub CLI(GitHub をコマンドで操作する道具)

ターミナルに以下を貼り付けて Enter してください。

**Mac の場合**(1行ずつ順番に):

```
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

```
brew install gh
```

※ 1行目は「Homebrew」という道具のインストールです。途中でパスワード(パソコンのログインパスワード)を聞かれたら入力してください。画面に文字は出ませんが入力されています。

**Windows の場合**:

```
winget install --id GitHub.cli
```

インストールできたか確認します。以下を貼り付けて Enter:

```
gh --version
```

`gh version 2.x.x` のように表示されれば成功です。

#### 4-2. Claude Code(Claude をターミナルから使う道具)

公式の案内(https://docs.claude.com/en/docs/claude-code/overview)に従ってインストールしてください。多くの場合は次の1行です:

```
npm install -g @anthropic-ai/claude-code
```

> `npm: command not found` と出た場合は、先に Node.js(https://nodejs.org)をインストールしてから再実行してください。

### 5. GitHub と接続する(初回のみ)

ターミナルに以下を貼り付けて Enter:

```
gh auth login
```

質問がいくつか出ます。すべて**そのまま Enter**(GitHub.com → HTTPS → ブラウザでログイン)で大丈夫です。最後にブラウザが開くので、画面の8桁コードを入力して「Authorize」を押してください。

### 6. リポジトリを自分のパソコンに持ってくる

手順2で作った自分のリポジトリを、パソコンにコピー(clone)します。`あなたのユーザー名` と `リポジトリ名` は自分のものに置き換えてください。

```
gh repo clone あなたのユーザー名/my-health-manager
```

```
cd my-health-manager
```

> 💡 `cd` は「このフォルダの中に入る」という意味のコマンドです。

### 7. 初期設定スクリプトを実行する

以下を貼り付けて Enter:

```
bash scripts/setup.sh
```

これで、記録に使うラベル(分類タグ)の作成や、今月のデータファイルの準備が自動で行われます。「セットアップ完了」と表示されたら、続けて:

```
git add -A && git commit -m "chore: initial setup" && git push
```

> もし「Public です」というエラーで止まった場合は、リポジトリが公開設定になっています。「困ったときは」を見て Private に変更してください(あなたの健康情報を守るための安全装置です)。

### 8. Claude を起動して、最初の計画を作る

いよいよ本番です。ターミナルで(`my-health-manager` フォルダの中にいる状態で):

```
claude
```

Claude が起動したら、日本語でこう話しかけてください:

```
今月のメニューを作って
```

あとは Claude が選択式の質問(目標・トレーニング練度・頻度・場所・食事スタイルなど)をしてくれるので、答えていくだけです。最後に、1ヶ月分の計画と毎日の記録ページ(Issue)が自動で作られます。

> 体脂肪率がわからなければ「わからない」でOKです(未入力でも進みます)。

### 9. 毎日の使い方

| いつ | すること | 例 |
|---|---|---|
| トレーニング/食事のあと | Claude に結果を伝える(文章・アプリのスクショ・CSV なんでもOK) | 「今日はベンチプレス60kgを8回3セット。昼はコンビニのサラダチキン」 |
| 毎晩 | 「**本日のメニュー終了**」と伝える | グラフ付きの日次レポートが作られ、その日の Issue が閉じられます |
| 毎週日曜 | 「**今週のメニュー終了**」と伝える | 週間のふりかえりレポート(部位別ボリューム・体重推移など)が作られます |
| 翌月1日 | 「**今月のメニューを作って**」と伝える | 前月の実績を踏まえた新しい計画が作られます |

Claude の起動方法は毎回同じです: ターミナルを開く → `cd my-health-manager` → `claude`

各日の記録ページ(Issue)には、その日の目標(歩数・睡眠時間・水分量・食事・トレーニング)と、実績記入欄(トレーニング・食事・メトリクス・体調チェック・自分自身への言葉)が用意されています。体調チェックは「チェック=問題あり」で、痛みがある場合は無理をせず専門家に相談してください。

### 10. 困ったときは

**Q. 間違えて Public(公開)でリポジトリを作ってしまった**
ブラウザでリポジトリのページを開く → Settings → 一番下の「Danger Zone」→「Change repository visibility」→ Private に変更。

**Q. `gh: command not found` と出る**
手順4-1のインストールが済んでいません。ターミナルを一度閉じて開き直してから `gh --version` を試し、ダメなら 4-1 をやり直してください。

**Q. `permission denied` や「認証エラー」と出る**
`gh auth login` (手順5)をやり直してください。

**Q. コマンドを打つ場所(フォルダ)がわからなくなった**
ターミナルで `cd my-health-manager` と打てば、リポジトリのフォルダに移動できます。「そんなフォルダはない」と言われたら `cd ~` で最初の場所に戻ってから再度試してください。

**Q. Claude が変な種目・知らない食材を提案してきた**
仕様上、リポジトリ内のデータベース(出典付き)にある種目・食材しか提案されません。もし DB にないものが出てきたら不具合なので、テンプレート配布元の Issues で報告してもらえると助かります。

**Q. スマホからも使いたい / ターミナルをどうしても使いたくない**
Claude.ai(ブラウザ/アプリ)+ GitHub 連携でも利用できます。設定手順が少し異なるため、この README の「Claude との接続 > Claude.ai(Web / モバイル)」の項を参照してください。

**Q. 途中で何が起きているか不安になった**
すべての計画・記録・レポートは、ブラウザで自分のリポジトリのページを開けば確認できます。「Issues」タブ = 毎日の記録、「Code」タブの `plans/` = 計画、`results/` = レポートです。

### 用語ミニ辞典

- **リポジトリ**: GitHub 上の、あなた専用の保存フォルダ
- **Issue(イシュー)**: リポジトリに付いているメモページ。ここでは「1日1ページの記録用紙」として使う
- **Private / Public**: 非公開/公開。健康情報を扱うため必ず Private
- **clone(クローン)**: リポジトリを自分のパソコンにコピーすること
- **ターミナル**: パソコンに文字で指示を出すアプリ。コピー&ペーストと Enter だけ使えればOK

## できること(月次サイクル)

トレーニングの PDCA と同型で回します。月初の計画作成が「プログラム設計」、日次 Issue が「トレーニングログ」、週次レポートが「デロード判断」、月次振り返りが「メゾサイクル評価」に相当します。

1. **毎月1日**: Claude に「今月のメニューを作って」→ 選択式ヒアリング(目標・トレーニング練度・頻度など)→ 計画合意 → 当月全日分の日次 Issue が一括発行される
2. **毎日**: 当日 Issue に結果を記載(手動 / テキスト・スクショ・アプリ出力の CSV/PDF を Claude に渡して代筆)
3. **毎晩**: 「本日のメニュー終了」→ グラフ付き日次レポート生成・Issue クローズ
4. **毎週日曜**: 「今週のメニュー終了」→ 週次レポート(部位別ボリューム・増減率・体組成推移)
5. **翌月1日**: 前月実績を自動で読み込み、次月計画に反映

## Claude との接続

### Claude Code(CLI / デスクトップ)

リポジトリ直下で起動するだけで `CLAUDE.md`(行動規範)と `.claude/` の Skills・Agents が自動読込されます。gh CLI の認証(`repo` スコープ最小限。Projects 連携する場合は `project` スコープ追加)が必要です。

### Claude.ai(Web / モバイル)

1. `bash scripts/package_skills.sh` で各スキルを zip 化(`dist/` に出力)
2. Settings > Features から各 zip をアップロード(コード実行有効な Pro/Max/Team/Enterprise が前提)
3. Project を作成し、`CLAUDE.md` の内容を Project instructions に転記
4. GitHub MCP コネクタを接続し、**認可対象を自分のこのリポジトリのみに限定**

## GitHub Projects(任意)

月次ボード(カラム: `Todo / Done / Skipped`)を自分のアカウントに作成すると、Issue 発行時に自動で紐付けられます。作成しなくても `month:YYYY-MM` ラベルだけで月次集計は完結します。

## 事実性の担保(このシステムの設計思想)

- メニュー・食事は `references/` の種目DB(78種目)・食材DB(52食材)**のみ**から生成されます。出典・実在確認済み参考URL付き
- トレーニングメニューはヒアリングで確認した練度(初心者/中級者/上級者)に応じて、種目の難易度・ボリューム・分割が調整されます
- 実績に DB 外の種目・食材を記録すると、確認の上 DB へ追加登録されます(DB追随更新)
- アプリ出力ファイル(CSV/PDF/HTML)はファイル記載の値のみで取り込まれ、列の対応は初回にあなたが確認します
- ⚠️ 食材DBの栄養値は日本食品標準成分表ベースの参考値です。**運用開始前に `references/nutrition-db.md` を確認することを推奨します**

- 検証済みの種目・食材をテンプレートへ還元する場合は [CONTRIBUTING.md](CONTRIBUTING.md)

## 免責

本システムは医療行為・診断・治療を提供しません。急激な減量目標・傷病の申告・体調不良の記録を検知した場合、計画生成は保留され、専門家(医師・管理栄養士・トレーナー)への相談を促します。

## プライバシー原則

- リポジトリは Private 必須(体組成・健康情報を含むため)
- スクリーンショット・アプリ出力に氏名等が写る場合はトリミング推奨(氏名等は Issue へ転記されません)
- GitHub MCP の認可は本リポジトリのみに限定
- gh CLI のトークンは最小スコープ
